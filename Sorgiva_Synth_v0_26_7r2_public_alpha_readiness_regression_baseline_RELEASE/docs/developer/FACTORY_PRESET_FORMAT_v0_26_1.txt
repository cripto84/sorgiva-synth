Regola browser/tassonomia
Ogni preset deve avere una categoria che il browser può mostrare e filtrare. Le categorie orfane devono restare 0.

Regola Arp/Sequencer nei factory
Nella linea attuale Arpeggiator e Step Sequencer sono OFF di default nei factory preset. I preset possono essere motion-ready, ma non devono attivare automaticamente arp/seq salvo scelta futura esplicita e documentata.

Campi tipici
Un factory preset dovrebbe avere:
- id stabile e unico;
- name leggibile e unico;
- category coerente;
- description utile;
- tags se presenti;
- parameters con soli parametri sonori/performance-musicali validi.

Cosa non mettere in parameters
- scope-enabled;
- spectrum-enabled;
- filtri UI del browser preset;
- stato tab;
- MIDI Learn runtime;
- MIDI Channel Filter runtime;
- metadati duplicati come category/originalCategory dentro parameters;
- dati diagnostici.

Aggiungere un futuro preset factory
Procedura consigliata:
1. Creare pochi preset alla volta.
2. Controllare che sfruttino davvero Sorgiva Synth e non siano cloni deboli.
3. Usare nomi chiari.
4. Mettere category esistente oppure aggiungere categoria nuova nella tassonomia e nel browser.
5. Verificare ID e nome unici.
6. Testare caricamento.
7. Testare volume/gain guard.
8. Verificare che Arp/Seq restino OFF se è factory standard.
9. Aggiornare count e documentazione.

Aggiungere una nuova categoria
Procedura minima:
1. Decidere se è davvero necessaria.
2. Aggiungerla alla tassonomia in preset-constants.js.
3. Assicurarsi che il browser la mostri.
4. Assegnare preset reali alla categoria.
5. Controllare categorie orfane = 0.
6. Aggiornare guide utente e sviluppatore.

Regola qualità
Meglio 12 preset forti che 50 mediocri. La factory library deve essere utile, non gonfia.
