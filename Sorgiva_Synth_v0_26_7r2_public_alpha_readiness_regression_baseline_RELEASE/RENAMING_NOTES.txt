Sorgiva Synth — Renaming Notes
==============================

Date: 2026-06-24
Build: v0.26.4c — LocalStorage / User Data Compatibility Migration
Previous public/internal name: SynthX Rebuild
New public name: Sorgiva Synth

Reason
------

The project is being renamed before public alpha to avoid public confusion with existing commercial synthesizer products using very similar naming.

Rules from this build onward
----------------------------

- Public UI name: Sorgiva Synth.
- Public documentation name: Sorgiva Synth.
- Official release bundle prefix: Sorgiva_Synth.
- Historical references may still mention SynthX Rebuild when describing builds up to v0.26.3f.
- `window.SorgivaSynth` is now the official runtime namespace.
- `window.SynthX*` aliases are intentionally retained as compatibility aliases, not as public branding.
- Legacy preset/user-bank metadata fields and import compatibility may keep recognizing `synthxVersion`, `.synthx-arp`, `.synthx-pattern` and old localStorage keys.
- New exports should use Sorgiva Synth names where user-visible.
- New export metadata should include `sorgivaVersion` / `sorgivaSynthVersion` and keep `synthxVersion` only as a legacy compatibility field.

No audio/sound change
---------------------

This rename/refactor line does not intentionally change audio engine behavior, preset sound design, MIDI, Step Sequencer, Arpeggiator, Mod Matrix, Randomizer, Morph, A/B Compare or factory preset count.

Next recommended step
---------------------

v0.26.4c — LocalStorage / User Data Compatibility Migration; v0.26.4d — Residual Branding Cleanup QA; v0.26.5 — Public Alpha QA Candidate.


v0.26.5 candidate note
----------------------
Sorgiva Synth v0.26.5 is the first Public Alpha QA Candidate after the rename. Legacy SynthX globals, JSON fields, schemas and localStorage keys remain intentionally supported for compatibility and migration.
