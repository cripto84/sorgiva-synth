Sorgiva Synth — Security Policy
================================

Supported status
----------------

Sorgiva Synth is still in pre-public-alpha preparation. The current mainline is a desktop/core web app.
Mobile/tablet is planned as a separate future fork and is not covered by the desktop/core release promise.

Reporting security issues
-------------------------

Until a public repository/contact channel is finalized, report serious issues through the official project
maintainer channel used for Sorgiva Synth development. When a public repository exists, use its preferred private
security contact or issue policy.

Please do not publicly post exploit details for active vulnerabilities before the maintainer has had a reasonable
chance to assess and fix them.

Scope
-----

Relevant issues include, for example:

- unsafe file import behavior;
- malicious preset/import vectors;
- browser storage misuse;
- unexpected network access;
- dependency or third-party code risks if dependencies are added in future.

Current technical notes
-----------------------

- Sorgiva Synth is currently a local/static WebAudio app with no intentional external network calls.
- Preset import/export uses JSON and should remain data-only.
- MIDI, audio and browser permissions are runtime features controlled by the browser and operating system.
